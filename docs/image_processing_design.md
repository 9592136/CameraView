# 显微图像处理算法设计

本文档描述伪彩、荧光通道融合、荧光染料管理、图像拼接和 EDF 景深扩展的第一阶段算法策略。它是实现前的设计约束，不代表已经进入代码实现。

## 1. 总体原则

- 原始图像只读保存，伪彩、融合、拼接和 EDF 都生成派生图像。
- 派生图像必须保留源图引用和处理参数，便于复查。
- UI 不直接执行耗时像素计算，拼接和 EDF 通过 `ProcessingQueue` 后台执行，并由 `ProcessingJobExecutor` 统一生成处理结果。
- 图像处理算法封装在 `imaging/`，应用流程封装在 `app/`，项目保存封装在 `storage/`。
- 后续可在 `imaging/` 内部接入 OpenCV 等开源库，但领域对象不暴露第三方库类型。

## 2. 伪彩显示

| 项目 | 第一阶段策略 |
| --- | --- |
| 输入 | 单通道灰度图或相机灰度帧 |
| 参数 | 强度最小值、强度最大值、显示颜色或 LUT |
| 输出 | RGB 显示图像 |
| 数据保存 | 保存原图引用、强度范围、颜色/LUT 名称 |
| 约束 | 不修改原始像素数据 |

推荐流程：

1. 将输入像素按强度范围归一化到 0-1。
2. 小于最小值截断为 0，大于最大值截断为 1。
3. 使用单色映射或 LUT 生成 RGB。
4. 将结果作为显示帧传给 `ImageViewport`。

## 3. 荧光通道融合

| 项目 | 第一阶段策略 |
| --- | --- |
| 通道数量 | 2-4 个通道 |
| 通道模型 | `FluorescenceChannel` |
| 配方模型 | `ChannelFusionRecipe` |
| 混合方式 | 加法融合，必要时归一化防止过曝 |
| 结果 | RGB 融合显示图像 |

每个通道记录：

- 源图引用。
- 染料或自定义颜色。
- 可见性。
- 强度范围。
- 通道透明度或权重。

融合时只合成显示图像，不覆盖任意单通道源图。

## 4. 荧光染料管理

`DyeProfile` 第一阶段字段：

| 字段 | 说明 |
| --- | --- |
| `id` | 稳定唯一标识 |
| `name` | 染料名称 |
| `excitationNm` | 激发波长 |
| `emissionNm` | 发射波长 |
| `displayColor` | 默认显示颜色 |
| `notes` | 备注 |

第一阶段先支持用户自定义染料。若后续需要预置库，可在 `DyeRepository` 初始化时载入常见染料列表。

## 5. 图像拼接

| 项目 | 第一阶段策略 |
| --- | --- |
| 输入 | 多张有重叠区域的图像 |
| 位置来源 | 文件导入顺序、手动位置提示或后续载物台位置 |
| 关键步骤 | 预处理、特征/相位相关配准、整体位姿优化、接缝融合 |
| 输出 | 全景派生图像 |
| 后台任务 | 通过 `ProcessingQueue` 执行，`ProcessingJobExecutor` 负责生成统一作业结果 |

第一阶段建议先支持手动导入多图拼接，不强依赖电动载物台。若配准失败，保留手动调整控制点或重叠比例的扩展点。

## 6. EDF 景深扩展

| 项目 | 第一阶段策略 |
| --- | --- |
| 输入 | 同一视野不同 Z 层的图像序列 |
| 数据模型 | `ImageStack` |
| 关键步骤 | 序列对齐、清晰度评分、焦点图生成、清晰区域合成 |
| 输出 | `EdfResult`，包含合成图和焦点图 |
| 后台任务 | 通过 `ProcessingQueue` 执行，`ProcessingJobExecutor` 负责生成统一作业结果 |

第一阶段先支持导入 Z 序列图片。电动 Z 轴联动采集放到后续阶段，避免过早绑定硬件接口。

## 7. 项目保存要求

项目元数据应保存：

- 原始图像文件引用。
- 派生图像文件引用。
- 伪彩和融合参数。
- 染料资料或染料引用。
- 拼接图块、位姿变换和融合参数。
- EDF 源序列、清晰度图和合成图引用。
- 测量实体、标定信息和测量结果。

这样可以保证导出的结果能回溯到原始图像和处理参数。
