# 真实相机现场预览验证清单

本文档用于连接 MUCam 相机后验证实时预览、刷新稳定性和基础交互。自动测试可以证明核心算法和项目读写，真实预览仍需要在有相机和 SDK 驱动的电脑上现场确认。

## 1. 准备

- 使用与相机驱动一致的平台运行程序：32 位驱动选 `Win32`，64 位驱动选 `x64`。
- 确认输出目录中已有 `MUCam32Ex.dll` 或 `MUCam32.dll`。
- 连接相机并确认系统可以识别设备。
- 关闭其他占用同一相机的软件。

## 2. 启动和枚举

1. 启动 `CameraView.exe`。
2. 点击 `Refresh`。
3. `Device` 下拉框应列出至少一个设备。
4. 选择目标设备后点击 `Open`。

通过标准：
- 未打开预览前，状态栏右侧显示已加载的 SDK DLL、接口类型和可选能力，例如 `SDK MUCam32Ex.dll | Ex | Exposure | AutoExp | Gain | WB | Bayer | RGB`。
- 状态栏左侧显示正在预览的设备、相机类型和分辨率。
- 状态栏右侧显示类似 `Device 1 | type ... | 1920x1080 | 25.0 fps | ts ...` 的实时遥测。

## 3. 预览稳定性

连续观察 3 分钟：

- 图像持续刷新，没有明显整窗闪烁。
- 状态栏右侧 FPS 每秒更新，数值没有长期掉到 0。
- 时间戳持续变化。
- 程序窗口可以移动、缩放，UI 不明显卡顿。
- 没有出现 `Camera disconnected` 或连续抓帧失败提示。

## 4. 基础交互

- 在 Camera 卡中点击 `Auto Exposure`、调整 `Exposure` 和 `Gain` 后点击 `Apply`，再点击 `White Balance`，画面亮度和色彩应有响应；不支持的相机应显示不可用提示。
- 使用鼠标滚轮缩放图像，光标附近图像位置应稳定。
- 使用右键或鼠标中键拖动放大后的图像，画面应平滑平移。
- 切换伪彩模式，显示效果应变化，原始采集不应中断。
- 做一次两点标定和长度测量，测量叠加线应跟随画面正确绘制。
- 点击 `Save Diagnostic`，保存现场诊断报告。

## 6. 常见问题

| 现象 | 检查项 |
| --- | --- |
| `SDK DLL not loaded` | 检查输出目录是否有 MUCam SDK DLL，平台是否和 DLL 位数一致 |
| `No MUCam camera found` | 检查 USB/供电/驱动，确认没有其他软件占用相机 |
| `Failed to open camera` | 重新插拔设备，关闭占用相机的软件，重新点击 `Refresh` |
| FPS 很低 | 检查曝光时间、USB 接口、分辨率和系统负载 |
| 画面颜色异常 | 确认 Bayer 转换函数是否由当前 SDK DLL 导出 |
