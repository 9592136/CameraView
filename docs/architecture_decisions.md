# 显微观察与测量软件架构决策记录

本文档记录当前设计阶段已经形成并已被用户确认的关键架构决策。若后续用户确认变更，应在这里追加新的决策记录，而不是直接覆盖历史背景。

## ADR-001 第一阶段继续使用 Win32 原生界面

| 项目 | 内容 |
| --- | --- |
| 状态 | 已确认 |
| 决策 | 第一阶段继续使用现有 Win32 原生窗口，后续再评估 Qt |
| 原因 | 当前项目已基于 Win32 和 MUCam SDK 工作，继续沿用可以先稳定相机、预览、测量和图像处理边界 |
| 影响 | 需要把 `main.cpp` 中的窗口逻辑逐步抽到 `ui/`，但不在第一阶段做 UI 框架迁移 |

## ADR-002 使用 MSVC + CMake 作为第一阶段构建路线

| 项目 | 内容 |
| --- | --- |
| 状态 | 已确认 |
| 决策 | 使用 Visual Studio C++/MSVC 编译器和 CMake 管理工程 |
| 原因 | 当前环境和 Windows 相机 SDK 更贴近 MSVC；CMake 便于后续拆分多模块 |
| 影响 | 新增模块应进入 `CMakeLists.txt`，保持可构建状态 |

## ADR-003 通过 `ICameraDriver` 隔离相机 SDK

| 项目 | 内容 |
| --- | --- |
| 状态 | 已确认 |
| 决策 | UI 和应用层只依赖 `ICameraDriver`，MUCam 细节放入 `MUCamCameraDriver` |
| 原因 | 便于后续更换相机 SDK 或增加模拟相机驱动 |
| 影响 | `MUCamApi` 保留为底层适配，不向测量、图像处理和 UI 暴露 SDK 句柄 |

## ADR-004 测量统一使用图像坐标

| 项目 | 内容 |
| --- | --- |
| 状态 | 已确认 |
| 决策 | 测量点保存图像像素坐标，屏幕坐标只用于交互和绘制 |
| 原因 | 缩放、平移、窗口尺寸变化不应改变测量结果 |
| 影响 | `ViewTransform` 负责屏幕坐标和图像坐标转换，测量实体不直接接触 UI 坐标 |

## ADR-005 原始图像和派生图像分离保存

| 项目 | 内容 |
| --- | --- |
| 状态 | 已确认 |
| 决策 | 原始图像只读保存，伪彩、融合、拼接、EDF 生成派生图像 |
| 原因 | 显微图像需要可追溯，处理结果应能回到源图和参数 |
| 影响 | `ProjectDocument` 需要保存源图引用、派生图引用和处理参数 |

## ADR-006 拼接和 EDF 使用后台任务队列

| 项目 | 内容 |
| --- | --- |
| 状态 | 已确认 |
| 决策 | 拼接和 EDF 通过 `ProcessingQueue` 执行，UI 只接收进度、取消、失败和完成通知 |
| 原因 | 大图和 Z 序列处理可能耗时，不能阻塞实时预览和鼠标交互 |
| 影响 | `StitchingService` 和 `EdfService` 提交 `ProcessingJob`，处理完成后发布派生图像到会话 |

## ADR-007 第一阶段项目文件使用 JSON 元数据 + 图像文件

| 项目 | 内容 |
| --- | --- |
| 状态 | 已确认 |
| 决策 | 项目元数据使用 JSON 保存，源图和派生图单独保存为 BMP/PNG 等图像文件 |
| 原因 | 结构清晰，便于调试和后续版本升级 |
| 影响 | `ProjectRepository` 管理元数据，`ImageExporter` 管理图像文件，导出服务统一编排 |
